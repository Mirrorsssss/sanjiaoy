## 1. 文档概述

### 1.1 对接目标
通过自动化对接权威游戏数据源，实现三角眼工具站核心业务数据（交易行物价、制造利润、子弹兑换、智能配装、地图点位）的自动同步，确保核心数据准确率≥99%，数据更新延迟≤10分钟。

### 1.2 核心对接范围
- 交易行实时物价自动同步
- 工坊制造利润自动化计算
- 子弹兑换最优方案自动匹配
- 按场景/预算的智能配装生成
- 物品历史价格走势与涨跌预警
- 全地图点位与机制数据同步

## 2. 数据源优先级与通用对接规范

### 2.1 数据源优先级
| 优先级 | 数据源类型 | 核心用途 | 稳定性 |
|--------|-----------|----------|--------|
| 第一优先级 | 三角洲数据帝开放平台 | 全量核心业务数据的主数据源 | 极高（已被好游快爆、TapTap等主流平台验证） |
| 第二优先级 | 腾讯官方公开数据 | 地图点位、版本更新公告、基础数值校验 | 极高（官方原生数据） |
| 第三优先级 | 开源社区项目 | 数据备份、技术参考，不做主数据源 | 中 |

### 2.2 通用对接规范
1. 所有API请求统一使用HTTPS协议
2. 所有请求头必须添加统一标识：`User-Agent: SanJiaoYan/1.0 (https://df.sanjiaoy.top)`
3. 免费API请求频率限制：单接口请求≤1次/秒，批量请求≤100次/分钟
4. 所有页面必须在底部标注数据来源：`数据来源：三角洲数据帝(orzrice.com) | 三角洲行动官方`
5. 必须建立多层数据缓存机制，避免重复请求上游服务器
6. 所有数据同步必须记录更新时间，前端页面需明确展示数据最新更新时间

## 3. 核心数据源：三角洲数据帝开放平台API接口大全

### 3.1 授权说明
- **Token申请地址**：https://orzrice.com/workApi/token_apply （待确认，需访问官网获取）
- 所有接口需携带授权Token
- Token可通过平台官方渠道免费获取，无调用量限制
- 仅要求标注数据来源

### 3.2 必接核心API接口

#### 3.2.1 实时交易行物价接口
- **接口地址**：`https://orzrice.com/workApi/v1/sjz_api/item_list`
- **请求方法**：GET
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| token | string | 是 | 授权Token |
| type | string | 否 | 物品类型。可选值：all(全部), weapon(武器), armor(护甲), ammo(子弹), material(材料) |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认50，最大100 |

- **请求示例**：
  ```http
  GET https://orzrice.com/workApi/v1/sjz_api/item_list?token=YOUR_TOKEN&type=material&page=1&page_size=50
  ```

- **返回值格式**：
  ```json
  {
    "code": 200,
    "msg": "success",
    "data": [
      {
        "item_id": "1001",
        "item_name": "钛合金板",
        "item_type": "material",
        "price": 2280,
        "price_min": 2200,
        "price_max": 2350,
        "update_time": "2026-05-12 18:30:00",
        "trend": "up",
        "is_bindable": false
      }
    ],
    "total": 1256,
    "page": 1,
    "page_size": 50
  }
  ```

- **更新频率**：每5分钟
- **落地建议**：每10分钟全量同步一次，存入本地缓存，前端读取缓存数据

## 4. 备用数据源：腾讯官方公开数据对接方案

### 4.1 官方版本更新公告RSS
- **地址**：`https://df.qq.com/news/rss`
- **用途**：自动监控游戏版本更新，提前触发数据同步流程
- **对接方式**：使用RSS解析库定时抓取
- **更新频率**：实时

### 4.2 官方地图工具
- **地址**：`https://df.qq.com/cp/a20240930tools/m/index.html`
- **可获取数据**：全地图物资点位、撤离点、首领位置、特殊机制
- **对接方式**：抓取页面静态JSON数据，转换为本地地图系统可用格式
- **更新频率**：游戏版本更新时同步

### 4.3 官方小程序接口（用户授权类）
- **基础地址**：`https://df.qq.com/mini/api`
- **可获取数据**：用户个人战绩、背包物品、交易行订单
- **对接方式**：引导用户扫码登录官方小程序，获取用户access_token
- **使用限制**：仅用于个人用户查询，不得批量爬取

## 5. 数据校验与异常处理方案

### 5.1 多数据源交叉校验机制
1. 所有核心数据（物价、制造利润）同时从主数据源和官方数据源获取
2. 当两个数据源数值差值≤5%时，使用主数据源数据
3. 当差值>5%时，触发数据异常标记，24小时内完成人工校验
4. 建立数据波动报警规则，当单物品价格24小时内波动超过20%时自动标记异常

### 5.2 数据源降级策略
| 业务场景 | 主数据源 | 第一备用 | 第二备用 | 最终降级方案 |
|----------|----------|----------|----------|--------------|
| 实时物价 | 三角洲数据帝API | 官方小程序接口 | 开源API | 显示最后一次正确数据 |
| 制造利润 | 三角洲数据帝API | 本地标准计算公式 | 静态推荐数据 | 显示静态推荐 |
| 地图点位 | 官方地图工具 | 三角洲数据帝API | 静态地图 | 显示静态地图 |
| 智能配装 | 三角洲数据帝API | 本地配装规则 | 静态配装方案 | 显示静态配装 |

### 5.3 接口错误码处理
| 错误码 | 含义 | 自动处理方式 |
|--------|------|--------------|
| 401 | Token无效 | 自动触发Token重新获取流程 |
| 429 | 请求频率超限 | 自动降低请求频率，增加重试间隔 |
| 500 | 上游服务器错误 | 自动重试3次，仍失败则切换备用数据源 |
| 503 | 上游服务维护 | 自动切换备用数据源，每30分钟检查恢复情况 |

## 6. 对接实施阶段规划

### 6.1 紧急修复阶段（1-3天）
1. 完成主数据源授权Token申请（访问 https://orzrice.com ）
2. 对接实时交易行物价、制造利润计算2个核心接口
3. 完成数据自动同步与缓存机制开发
4. 修复现有页面的核心数据错误

### 6.2 短期优化阶段（4-7天）
1. 对接子弹兑换、智能配装、历史价格3个核心接口
2. 完成对应前端页面的功能开发与数据渲染
3. 对接官方地图工具，完成静态地图到半互动地图的升级
4. 建立数据异常报警与自动降级机制

### 6.3 中期功能开发阶段（8-30天）
1. 对接WebSocket实时推送接口，开发价格提醒功能
2. 对接官方小程序授权接口，开发个人战绩查询功能
3. 开发配装分享、收藏功能
4. 完成全量数据的自动化校验与定时同步机制开发

### 6.4 长期稳定阶段（31-90天）
1. 建立完整的数据监控与运维体系
2. 每季度完成一次全量数据校验与优化
3. 跟随游戏版本更新，同步完成功能迭代与数据更新
4. 优化用户体验，提升页面加载速度与响应效率

## 7. 注意事项
1. 严格遵守各数据源的使用规范，禁止进行恶意爬取、高频请求
2. 所有数据必须在页面明确标注来源
3. 必须建立每日数据备份机制
4. 游戏版本更新前，提前完成数据同步准备
5. 定期审查数据准确性，每季度完成一次全量数据交叉校验
6. 所有接口请求必须做好异常捕获
