交接指南 新开发者必读

🔧 日常维护流程

  1. 修改数据 → 直接编辑对应 JSON 文件（workshop.json / price-realtime.json 等）
  2. 修改 UI → 编辑对应 .html 文件，保持与现有 CSS 变量一致（主色 #ff6b00、背景 #0a0e1a）
  3. 提交时请使用规范 commit message：fix: 修复 / feat: 新功能 / docs: 文档
  4. 提交后必须更新 devlog.html：在对应日期分组下新增一条 commit entry（参考上方格式）
  5. 同步更新 changelog.html 的用户公告区（添加新日期条目）
  6. 推送到 GitHub：git push（自动触发 GitHub Pages 部署）
📌 Commit Message 规范

fix: 修复数据错误（如价格、兑换比例）
feat: 新增功能/页面/数据
docs: 文档更新（changelog/devlog）
refactor: 代码重构（不影响数据）

⚠️ 数据更新注意

price-realtime.json 的 is_get_time 字段需同步更新为最新日期。
工坊税率如有变动，修改 workshop.json 中的税率公式（当前为 13% = 10% + 3%）。
S9 赛季标识为 ff6b00 橙色主题。

📁 Devlog 更新步骤

每次 commit 后，在 devlog.html 的顶部当天日期分组下添加新条目：
① 复制 commit hash
② 填写 commit message
③ 列出所有改动的文件（标 new/modified）
④ 添加标签（feat/fix/s9/data/ui）
⑤ 更新页面顶部统计数字
